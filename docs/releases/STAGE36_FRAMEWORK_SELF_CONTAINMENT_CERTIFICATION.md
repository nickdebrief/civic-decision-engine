# Stage 36 — Framework Self-Containment Certification

## Purpose

Stage 36 certifies visible methodological self-containment for the Civic Record
Evaluation Framework. It determines whether CREF is sufficiently described by
its visible methodology, stages, phases, deterministic outputs, governance
structures, boundaries, and documentation to be understood separately from the
Civic Decision Engine implementation.

## Why Stage 36 Matters

CREF is the methodology. Civic Decision Engine is one implementation of that
methodology. Making this distinction inspectable allows CREF to be studied,
critiqued, and independently implemented in another institution, country, or
technical environment without treating the original software runtime as the
methodology itself.

This certification is deliberately narrow. It certifies visible methodological
separability only. It does not certify that a third party can implement CREF
correctly, that the software is portable, that external adoption is ready, or
that CREF is legally or institutionally valid in another context.

## Self-Containment Checks

The certification contains 35 stable checks covering:

- methodology description, purpose, scope, and limitation visibility;
- structural representation of evaluation and reflexive inspection phases;
- visibility of the governance lifecycle;
- report modes, dependency mapping, pathways, transitions, provenance, replay,
  integrity, audit packaging, conformance, closure, continuity, change
  registration, governance, lineage, and lifecycle review;
- methodology/implementation separation and Civic Decision Engine independence;
- cross-implementation interpretability and external-adoption boundaries;
- documentation dependency and open methodology visibility;
- future-stage non-inference, non-mutation, non-authority, and gap absence;
- independent methodology portability declaration and limitation visibility.

Each check records its identifier, name, category, affected stage or output,
declared and observed state, certification result, basis, and limitation.

## Methodology / Implementation Separation

The visible relationship model states:

- CREF methodology is distinct from Civic Decision Engine software.
- Civic Decision Engine implements CREF but does not define the entire
  methodology.
- CREF is described through declared stages, phases, limitations, governance
  outputs, and documentation.
- Independent implementation requires preserving declared methodology,
  boundaries, limitations, and deterministic principles.
- Self-containment concerns methodological portability, not software deployment
  portability.
- Future stages are not inferred.
- Certification uses visible framework outputs and declared metadata only.

## Relationship To Civic Decision Engine

Stage 36 does not reject or diminish Civic Decision Engine. It records the
implementation relationship accurately: the engine operationalises CREF, while
CREF remains describable as a methodology through its own declared structure.
The methodology still depends on its documentation; self-containment does not
mean independence from all explanatory material.

## Report Modes

- **Executive Report:** concise overview and key limitations.
- **Review Report:** overview, seven relationship declarations, and full
  limitations.
- **Full Inspection Report:** overview, relationships, all 35 checks, and full
  limitations.

Full Inspection remains the default report mode.

## Limitations

Framework Self-Containment Certification is not legal certification or external
accreditation. It does not validate evidence, determine truth or liability,
infer intent, infer future stages, or infer undocumented implementations. It
does not certify third-party correctness, software portability, institutional
adoption readiness, legal validity, external validity, or authority.

It does not modify records, classifications, thresholds, dependencies, evidence
relationships, transition history, provenance, replay outputs, integrity
checks, audit packages, certifications, continuity outputs, change-register
entries, governance principles, version-lineage entries, lifecycle review
items, or report modes. It does not write to the database or alter public API
behaviour.

It certifies visible methodological self-containment only.

## Preservation And Non-Mutation Guarantees

Stage 36 consumes existing outputs without changing them. Stage 21 report modes
and all Stage 22–35 preservation counts remain unchanged. The builder copies
visible inputs, returns deterministic certification data, and persists nothing.

No schema, migration, database, hash, upload/download, public API,
classification, threshold, dependency, evidence-relationship, or mutation
change is introduced.

## Validation

Validation covers deterministic generation, input non-mutation, required check
fields and names, overview counts, relationship declarations,
complete/partial/gap states, report-mode rendering depth, and preservation of
Stage 22–35 counts. Compilation, whitespace and conflict-marker checks, focused
admin tests, the full test suite, and desktop/mobile browser inspection are
required before completion.

## Implementation Summary

Stage 36 adds one deterministic self-containment builder and scoped
administrative renderers to the existing admin record evidence flow. No route,
persistence mechanism, external integration, schema field, software portability
mechanism, or evaluative classification is added.
