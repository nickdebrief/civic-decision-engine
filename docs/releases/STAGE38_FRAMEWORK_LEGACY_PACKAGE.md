# Stage 38 — Framework Legacy Package

## Purpose

Stage 38 assembles a visible, deterministic legacy package for the Civic Record
Evaluation Framework. It collects the declared methodology artefacts needed for
independent understanding, preservation, and future continuation without
performing evaluation or changing existing outputs.

## Why Stage 38 Matters

Stage 36 established visible methodological self-containment and Stage 37
declared stewardship responsibilities. Stage 38 packages those foundations with
the implemented evaluation, inspection, and governance outputs so the
methodology can be preserved independently of a particular software runtime.

## Legacy Package Items

The complete package contains 36 deterministic items:

- 30 legacy items included directly;
- 6 legacy items included with explicit limitations;
- 0 unavailable legacy items; and
- 0 legacy gaps.

The items cover core methodology statements, phase declarations, report modes,
Stages 22–37 inspection and governance outputs, methodology/implementation
separation, preservation requirements, adoption and future-stage boundaries,
non-mutation, non-authority, and limitation visibility.

## Methodology And Implementation Separation

CREF is the methodology. The Civic Decision Engine is an implementation of
CREF. The package preserves the visible methodology without requiring a
particular runtime, but it does not certify software portability, third-party
implementation correctness, institutional adoption, or external validity.

## Relationships

Eight deterministic relationships describe the methodology package,
implementation separation, documentation package, governance package,
preservation package, adoption boundary, future boundary, and visible legacy
source.

## Overview Metrics

The complete fixture preserves 38 implemented stages, three declared phases,
six version-lineage relationships, 28 lifecycle review items, 35
self-containment checks, 25 stewardship declarations, and every established
Stage 22–34 preservation count.

## Report Modes

- **Executive Report:** overview and concise limitations.
- **Review Report:** overview, eight relationships, and full limitations.
- **Full Inspection Report:** overview, relationships, all 36 package items,
  and full limitations.

Full Inspection remains the default mode, and Stage 38 appears immediately
after Stage 37.

## Limitations

The Framework Legacy Package is not legal ownership, accreditation, external
certification, evidence validation, or institutional authority. It does not
infer undocumented implementations or future stages, certify software
portability or implementation correctness, approve adoption, or modify records
and existing framework outputs. It packages visible methodology artefacts only.

## Preservation And Non-Mutation Guarantees

Stage 38 preserves all Stage 21–37 behavior and counts. It introduces no schema,
migration, database write, public API change, canonical or attachment hash
change, upload/download behavior, classification change, threshold change,
dependency change, evidence-relationship change, external retrieval, or record
mutation.
