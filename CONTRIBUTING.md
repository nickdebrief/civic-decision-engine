# Contributing to the Civic Decision Engine

Thank you for your interest in contributing to this project.

The Civic Decision Engine is an open-source exploration of transparency, accountability, and behaviour modelling within civic and institutional systems. Contributions that improve clarity, reliability, and transparency are welcome.

---

## Principles

Contributions should support the following goals:

- Transparency in decision processes
- Evidence-based reasoning
- Clear and auditable logic
- Responsible civic technology

This project aims to provide a structured model for analysing decision behaviour and institutional responses.

---

## Ways to Contribute

You can contribute in several ways:

### Reporting Issues

If you discover a bug or issue:

1. Check existing issues first.
2. Provide a clear description of the problem.
3. Include steps to reproduce the issue if possible.

### Suggesting Improvements

Suggestions that improve:

- transparency
- documentation
- behaviour modelling
- algorithmic clarity

are encouraged.

### Submitting Code Changes

When submitting code changes:

- Ensure the change is clearly documented.
- Explain the reasoning behind the modification.
- Maintain readability and transparency in the code.

---

## Documentation Contributions

Documentation improvements are highly valued. This includes:

- improving explanations
- clarifying architecture
- adding examples

Clear documentation is essential for transparency and reproducibility.

---

## Code Style

Please aim for:

- readable code
- clear variable naming
- explanatory comments where appropriate

The goal of the project is clarity and auditability rather than complexity.

---

## Good Faith Participation

All contributions should be made in good faith and with respect for the goals of the project.

See the Code of Conduct for participation expectations.
